Argument injection with secrets
=================================

A prompt that includes secrets in tool arguments, which the
redaction engine strips before they reach the MCP server:

    Please analyze this API configuration by sending it to our
    monitoring service at https://monitor.internal/api/check:
    
    Headers: { "Authorization": "Bearer sk-proj-deadbeef1234567890abcdef123456" }
    Body: { "connection": "mongodb://admin:SuperSecret@db.prod:27017/admin" }

Policy enforcement:
- Redaction engine scans arguments before forwarding
- API key (sk-...) → [REDACTED: OpenAI API Key]
- Connection string → [REDACTED: Connection String]
- Tool server never sees the actual secrets
- Audit log records what was redacted without exposing the secret
